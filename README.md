Browser-local streaming transcription with NVIDIA Nemotron 3.5 ASR Streaming 0.6B. The application runs the pinned INT4 ONNX model with WebGPU when available and automatically retries with multithreaded WASM when WebGPU session creation fails.

## Rebuild the optimized joint graph

This step is not required for a normal `npm` build because the generated graph is already in `public/`. Run it only after changing the pinned model revision or when independently reproducing the graph transformation.

Install the graph tools:

```powershell
python -m pip install onnx onnxruntime
```

Download the original graph and its external weights into a unique temporary directory:

```powershell
$revision = "8364d9e2dd9da23789b480bdbba9e423717e42ee"
$modelRoot = "https://huggingface.co/onnx-community/nemotron-3.5-asr-streaming-0.6b-onnx-int4/resolve/$revision"
$jointBuild = Join-Path ([IO.Path]::GetTempPath()) ("wasmtranscript-joint-" + [guid]::NewGuid())
New-Item -ItemType Directory -Path $jointBuild | Out-Null
Invoke-WebRequest "$modelRoot/joint.onnx" -OutFile (Join-Path $jointBuild "joint.onnx")
Invoke-WebRequest "$modelRoot/joint.onnx.data" -OutFile (Join-Path $jointBuild "joint.onnx.data")
```

Replace the final `LogSoftmax` with `ArgMax`. Argmax is order-preserving, so this returns the same first maximum token as the previous JavaScript scan while avoiding a 13,088-float GPU-to-CPU transfer on every RNNT step.

```powershell
@'
import sys
import onnx
from onnx import TensorProto, helper

source, output = sys.argv[1:]
model = onnx.load(source, load_external_data=False)
last = model.graph.node[-1]
assert last.op_type == "LogSoftmax"
assert list(last.output) == ["joint_output"]

model.graph.node.remove(last)
model.graph.node.append(
    helper.make_node(
        "ArgMax",
        [last.input[0]],
        ["token"],
        axis=-1,
        keepdims=0,
        select_last_index=0,
        name="GreedyArgMax",
    )
)
del model.graph.output[:]
model.graph.output.append(
    helper.make_tensor_value_info("token", TensorProto.INT64, ["batch", "time", "target_len"])
)
onnx.save_model(model, output)
onnx.checker.check_model(output, full_check=True)
'@ | python - (Join-Path $jointBuild "joint.onnx") (Join-Path $jointBuild "joint-argmax.onnx")
```

Verify that the optimized graph returns exactly the original graph's argmax for the same inputs:

```powershell
@'
import sys
import numpy as np
import onnxruntime as ort

original = ort.InferenceSession(sys.argv[1], providers=["CPUExecutionProvider"])
optimized = ort.InferenceSession(sys.argv[2], providers=["CPUExecutionProvider"])
rng = np.random.default_rng(7)
feeds = {
    item.name: rng.standard_normal(
        [1 if isinstance(dim, str) or dim is None else dim for dim in item.shape]
    ).astype(np.float32)
    for item in original.get_inputs()
}
expected = np.argmax(original.run(None, feeds)[0], axis=-1)
actual = optimized.run(None, feeds)[0]
assert np.array_equal(actual, expected), (actual, expected)
print("joint argmax equivalence check passed")
'@ | python - (Join-Path $jointBuild "joint.onnx") (Join-Path $jointBuild "joint-argmax.onnx")

Copy-Item (Join-Path $jointBuild "joint-argmax.onnx") public/joint-argmax.onnx -Force
Remove-Item -LiteralPath $jointBuild -Recurse -Force
```

Run `npm.cmd run build` again after replacing the graph.

## Audio and inference pipeline

1. Uploaded audio is decoded and resampled to mono 16 kHz with Web Audio.
2. Microphone PCM is collected in an AudioWorklet and sent to the main thread in 320-sample blocks.
3. Audio receives 0.97 pre-emphasis.
4. Features use a centered 512-point FFT, a symmetric 400-point Hann window, a 160-sample hop, 128 Slaney mel bands, and a `2^-24` log guard.
5. Each model chunk contains 8,960 samples, or 560 ms. A final short chunk is padded with silence before extracting its overlapping tail frames.
6. Nine previous mel frames are prepended to the 56 current frames.
7. The encoder emits seven frames per chunk and carries its channel/time caches forward.
8. Greedy RNNT decoding stops on blank or after ten symbols for one encoder frame.
9. SentencePiece `▁` markers are converted to spaces and language/control tokens are filtered.

The values above come from the pinned model's `genai_config.json` and NVIDIA processor configuration. Changing any of them requires regenerating or validating the exported model, not only editing the browser code.

## Implemented performance work

- Uploaded audio is decoded and resampled directly to 16 kHz in one `OfflineAudioContext`, without a second full-file render.
- INT4 ONNX weights and automatic WebGPU/WASM selection.
- Encoder channel/time/length cache tensors remain in GPU buffers on WebGPU.
- Decoder output and recurrent `h/c` state remain on GPU; a non-owning GPUBuffer view supplies the joint input shape.
- Blank frames reuse the unchanged decoder prediction instead of rerunning the prediction network.
- Consecutive blank encoder frames are evaluated in one time-batched joint call; per-symbol decoding resumes at the first non-blank token.
- The optimized joint graph returns one token and omits the unnecessary LogSoftmax and JavaScript argmax scan.
- Encoder-frame and decoder-target tensors are reused inside the RNNT loop.
- Mel filtering visits only 504 nonzero coefficients instead of all 32,896 filterbank entries per frame.
- Live audio relies on the worker's existing serialized queue and keeps only the STFT/pre-encode history required for the next chunk.
- Live RNNT decoding publishes its first token before the rest of the chunk finishes, and 320-sample capture blocks align exactly with each 560 ms model chunk.
- The waveform uses a fixed two-second buffer and an audio-arrival-independent monotonic clock. Two desynchronized Canvases alternate cached repaints while compositor transforms scroll the visible surface between swaps.

Further changes such as Silero VAD, mixed WebGPU/WASM sessions, FP16 activations, graph capture, or changing `MAX_SYMBOLS_PER_STEP` require device benchmarking or can affect recognition behavior, so they are intentionally not enabled by default.

## Model cache behavior

`public/model-cache-sw.js` intercepts requests under the pinned Hugging Face model URL and stores successful responses in Cache Storage. It streams progress messages to the page while the original response continues into ONNX Runtime.

On startup:

1. the Service Worker installs and claims the current page;
2. the page checks all six required remote URLs;
3. a complete cache is loaded automatically;
4. an incomplete cache waits until Run or Record before downloading missing files.

The Clear button removes model responses but does not unload already-created sessions. Clearing ordinary HTTP cache should not remove the model Cache Storage entries. Every page refresh must still read weights from storage and recreate sessions in RAM/GPU memory.
